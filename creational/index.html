# 创建者模式

TIP

创建者模式提供了创建对象的机制， 能够提升已有代码的灵活性和可复用性。

工厂模式: 在父类中提供一个创建对象的方法， 允许子类决定实例化对象的类型。

抽象工厂模式: 创建一系列相关的对象， 而无需指定其具体类。

生成器模式: 允许使用相同的创建代码生成不同类型和形式的对象。

原型模式: 能够复制已有对象， 而又无需使代码依赖它们所属的类。

单例模式: 能够保证一个类只有一个实例， 并提供一个访问该实例的全局节点。
